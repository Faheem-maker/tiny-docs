# What's new in v1.0.4?
In this release, we implement several new tests, improve security and allow retaining older files during edit.

# Improved Coverage
While you will not see any changes (we hope), it's still an important step to ensure proper maintability in the long-term. We have added tests for both MySQL & Sqlite driver, ensuring that maximum coverage is reached.

Within the next 2-3 releases, you should expect to see about **80%** coverage, ensuring nothing breaks... ever.

# Improved Security
We have finally moved the webroot to `public` folder, significantly reducing attack surface and improving security. This might lead to a few issues during upgrade, but you can just drop a message in discussion if you happen to be using this framework.

[github.com/bolt-php/bolt/discussions](https://github.com/bolt-php/bolt/discussions)

# Bug Fixes
We have fixed several bugs, some of the major ones being the following:

* ORM unable to select data
* Path resolution on Linux
* Routes weren't named outside of a group

## Other Changes

The full list of changes is as follows:

### Fixes

* Fixed table existence check for SQLite driver
* Fixed duplicate component initialization bug
* Fixed ORM issue where data couldn’t be selected
* Fixed incorrect return type (array instead of object)
* Fixed path resolution issues on Linux
* Fixed nested parenthesis handling in `@if` directive
* Fixed handling of empty values in widget parameters
* Fixed route naming outside of groups

### Additions

* Added `Response::file()` for correct MIME type file responses
* Added `migration:rollback` command to revert migrations

### Changes

* Improved test coverage
* Modified `Model::fill` to ignore empty values
* Changed `serve` to start in the `public` directory
