# What's new in v1.0.2?
In this release, we include several new features, focusing on streamlining the minimum structure necessary to build a CRUD. Here's everything that we have changed in this release.

# File Uploads
It's now possible to upload files using an `ActiveModel`, or with a `Model`. Unfortunatley, there are no validations and customization options yet, but I shall include more options soon focusing on security for now.

## File Upload Example:
In order to upload a file, you need to define this model:

```php
class Product {
    public UploadedFile $image;
}
```

While filling the model, you would need to pass in `files` as well:

```php
$product = Product::from($request->post(), $request->files());

// Or with exisitng model
$product->fill($request->post(), $request->files());
```

Finally, the image can be displayed as follows:

```php
<img src="{{ $product->image->url() }}">
```

## Improved Scaffolding
The most important change is that you don't have to copy `.env` after starting a new project anymore. The file is auto-generated for you and `php tiny serve` runs out of the box.

You can also use `--rest` flag with `make:controller` command to auto-generate all REST actions while generating a new controller.

```bash
php tiny make:controller product --rest
```

## Other Changes
As always, we fixed multiple bugs, added a variety of small features and added more tests.

The full list of changes is as follows:
### Changes
- You can now pass multiple parameters to `ActiveModel::from`
- DateTime handling for database now uses transformers
- `UrlManager::named` now returns a full URL
- `.env` will now be auto-copied instead of requiring manual intervention.

### Fixes
- Fixed a bug that prevented us from setting values on ActiveModel
- Fixed namespace in table & form widgets

### Additions
- Added transformers for extending `ActiveModel` with custom types
- Added `--rest` flag for REST controller generation
- Added `params` to `named` & `to` methods for URL generation with auto-parameterization
